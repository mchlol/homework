Form Basics

Explain what the form element is for and what two attributes it should always include.

The form element is used to collect data from a user, such as leaving comments on a blog, filling out a survey, signing up to an email list, or taking payment on an e-commerce site.
The form must include an action attribute to determine where the data is sent, usually a url, and a method element to determine how the data is sent - usually GET to retrieve data from the server, or POST to send data to the server.

Explain what form controls are at a high level.

Form controls are elements in the form that the user interacts with to provide specific data - these can be input elements such as number, email, or telephone numbers, or selection inputs where some data is provided for the user to choose from, such as radio buttons, checkboxes, ranges, and even color-pickers or dates.

What is the name attribute for?

The name attribute provides the data with a reference, so when the data is sent to the server it can be identified with the relevant label.

What are the three most common form controls you can use for allowing users to select predefined options?

  • checkbox, which allow the user to select one or more from a range of items, displayed as boxes to be checked.
  • radio, which provides a short selection of items displayed as circles for the user to choose only one of.
  • select, like radio the user can only select one item, but this is used for a longer list of options displayed as a dropdown menu.

What are the three types of buttons in html?

  • Reset - clears all the data a user has entered into the form's inputs.
  • Submit - submits the form's data to the provided action.
  • Anonymous - a generic button that can do anything, usually DOM related using JavaScript.

What are the two most challenging aspects of styling forms?

One is that most input elements use native browser styling, and it can be difficult to remove these default styles and re-style them while remaining consistent across different browsers (and be accessible).
The other is that some elements cannot be styled with CSS at all, such as color-picker or date-time-local and the only way around this is to use JavaScript (however, many libraries are available for this).
Another tricky thing to remember is that padding and margin affect the inside of the input elements instead of the space around them. Putting the inputs inside div or li elements can get around this.

Form Validation

What does the required validation do?

The required validation lets the user know the field must be filled in, or the form will not submit.

What validations can you use for checking text length?

minlength and maxlength

How can you validate the minimum and maximum of numeric inputs?

min and max

What can you use the pattern validation for?

Pattern validation via regex can be used to ensure the user enters the correct information, such as a valid credit card number, or creating a strong password.

What pseudo css selectors are available for styling valid and invalid inputs?

:valid and :invalid

SVG

What is the xmlns attribute?

xmlns stands for xml (extensible markup language) name-space, which specifies how the elements markup should be interpreted.

What are some situations where you wouldn't want to use SVG?

SVG is a vector format so perfect for simple lines and shapes such as icons, charts, line drawings, and text. It is not suitable for images with a lot of data such as photos.

What are the benefits of “inlining” your SVGs? What are the drawbacks?

Inlining SVG files means that the file can be accessed through the DOM with CSS and JavaScript, eg. to animate, or change colours. However, large SVG files can make your page load slowly, be harder to cache, and harder for a human to read your HTML.

Tables

What is a table?

A table is used to structure data into an easily readable and referenceable format.

Why is it a bad idea to use HTML Tables for page layout?

Using HTML tables for page layout has been deprecated because it causes confusion for screenreaders.

What are caption elements useful for?

Caption elements give a brief summary of what the data in the table is for, and can be read by a screenreader to give the user the option of reading all the data in the table or not.

What is the scope attribute?

The scope attribute organises the data in the table so that a screenreader can easily read out what piece of data relates to which headers. Another way to do this is specifically label each cell of data with all the relevant headers, but this can be tedious to write and is prone to error.
