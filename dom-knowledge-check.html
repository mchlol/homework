What is the DOM?

The DOM, or Document Object Model, is a tree-like representation of the contents of an html page. When the html code is parsed by the brower it is converted to the DOM. For example, the body is the parent of all the elements that display on the page, which are in turn children of the body. Any element that contains another element is it's parent. The elements are called "nodes".

How do you target the nodes you want to work with?

You can target a specific node using CSS selectors. To target an element using its ID, you can use document.querySelector("#container") or you can target all the elements that share a class by using document.querySelector(".wrapper"). You can also select a group of nodes by selecting all divs or paragraphs with document.querySelectorAll("div").
Another way to target nodes is by using relationship selectors, such as firstElementChild or lastElementChild.

How do you create an element in the DOM?

Create an element, such as a div, in the DOM with const container = document.createElement("div").

How do you add an element to the DOM?

Add an element to the DOM by appending it to a div or container that you have assigned to a variable; 


    const container = document.querySelector("#container");
    const newDiv = document.createElement("div");
    newDiv.textContent = "I'm a new div!";
    container.appendChild(newDiv);

How do you remove an element from the DOM?

To remove an element you can use container.removeChild(newDiv).

How can you alter an element in the DOM?

You can add styles to your element directly through JavaScript, or by adding a class to it and then styling it with CSS.

When adding text to a DOM element, should you use textContent or innerHTML? Why?

You should use textContent, because allowing the innerHTML of an element to be altered leaves it vulnerable to malicious script injecting.

Where should you include your Javascript tag in your HTML file when working with DOM nodes?

Your Javascript file should go at the end of your body, so that it runs after all of the code in your HTML has been parsed by the browser. Otherwise it may not be able to access elements that have not been created yet.
You can link an external JavaScript file, or put a script tag at the end of your body.
You can also put it in your head tag, but use defer to run it last.
<script src="script.js" defer></script>

How do “events” and “listeners” work?

An event is a mouse click or hover, key press, or other form of interaction with an HTML page. A listener can 'listen' for an 'event' and run some code when the 'event' happens.

What are three ways to use events in your code?

  • Attach an event listener directly in the html tag of your element. Such as; <button onclick="alert('hello')">Say hello</button> =

  • You can also use the onclick property in an external javascript file.
    If your html is set up like so:
    <button id="btn2">Say hello</button>
    Your JavaScript file would access it like this: 
    const btn2 = document.querySelector("#btn2");
btn2.onclick = () => alert("hello...again");

    In either of the above two points, your button can only have one onclick property.

  • The third way is to use an event listener in your external JavaScript file, rather than just onclick. This way we can add any other kinds of event listeners to the button. 
    btn2.addEventListener('click', () => { 
                    alert("hello...again");
                });

Why are event listeners the preferred way to handle events?

we can attach an event listener for click, one for right click, one for hover, and so on.

What are the benefits of using named functions in your listeners?

Using a named function means you can read the code and understand what it does much easier, and you can reuse that function elsewhere.

How do you attach listeners to groups of nodes?

Attach a listener to a group of nodes, by using document.querySelectorAll("button") which would add the listener to all buttons in your html file. Iterate through the buttons (or other group of elements) with:


    buttons.forEach(button) => {
        button.addEventListener('click', () => {
            alert("Hello");
        });
    });

What is the difference between the return values of querySelector and querySelectorAll?

With querySelector you can select one element at a time, and it will only return that one element. With querySelectorAll, you can target all the elements of a specific type or class, and it will return them in a "nodelist".

Explain the difference between “capture” and “bubbling”.

'Bubbling' is when events are triggered upwards through the nodes of an element. When an event handler is triggered on an element, it will first happen on the element, then any handlers on it's parent with be triggered, and so on for any ancestors above it.
'Capturing' is when the events happen in the opposite order, or downwards. If an onclick property is attached to a paragraph, which is nested within a div, which in turn is a child of your body tag, any event listener on the body will happen first, then on the div, then finally on the paragraph.
This series of events is known as 'propogation' and is very useful for things like forms. However, it is possilbe to stop it with event.stopPropogation().
