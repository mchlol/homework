Intro to Git

What kind of program is Git?

Git is a version control program, which tracks the changes that have been made to files.

What are the differences between Git and a text editor in terms of what they save and their record keeping?

A text editor can only save one copy of a file at a time. So if you wanted to keep copies of your file at different stages, you would literally need several copies of it. With Git, a ‘snapshot’ of your code is copied, so if you need to revert your changes you only need to refer to to the snapshot to restore it.

Does Git work at a local or remote level?

Git works at a local level, but you can push your git repository (collection of files) to a remote location.

Does GitHub work at a local or remote level?

Github is a service where you can store the remote copy of Git repository.

Why is Git useful for an individual developer?

Git is useful for an individual developer as you can track all the changes you’ve made, and if necessary, revert back to a previous state.

Why are Git and GitHub useful for a team of developers?

Git and Github are useful for a team as multiple people can work on the same file at the same time, using branches, and then merge their changes together.

Git Basics

How do you create a new repository on GitHub?

On the GitHub website, click on the + sign at the top right and select new repository.

How do you copy a repository onto your local machine from GitHub?

With your repo open on the GitHub website, click on the ‘code’ button and copy the SSH option. Back at the command line, navigate to your project folder and enter git clone <SSH link>

What is the default name of your remote connection?

origin

Explain what origin is in git push origin main.

origin is the name of the remote connection to your repository on Github.

Explain what main is in git push origin main.

main is the main (formerly known as master) branch of your code.

Explain the two-step system that Git uses to save files.

git add adds files to staging, then git commit commits the files to the branch you’re working on.

How do you check the status of your current repository?

git status

How do you add files to the staging area in git?

git add <filename>

How do you commit the files in the staging area and add a descriptive message?

git commit -m “<essage>”

How do you push your changes to your repository on GitHub?

git push origin <branch name>

How do you look at the history of your previous commits?

git log

Commit Messages

What are two benefits of having well-written commit messages and a good commit history?

  1. Other developers will have a clear idea of what you have done with a file, and why.
  2. You may not always remember how or why you did something, so good commit messages also help you understand what you have done, when viewing a project after a long time away from it.
  3. Having a good commit history means you will have a 'save point' to return to if something goes wrong.

How many characters should the subject line of your commit message be?

The subject line should be less than 50 characters, with a hard limit of 72.
