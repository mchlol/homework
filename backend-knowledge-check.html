What is back-end development?

Back-end development is where all the data is stored for a website or web application to make it work. The front-end is where a user can interact with a website or app - but when a client requests information, the back-end provides it. The back-end can be written in any programming language, but languages may need to be installed on the server in order for it to be able to understand it.

How many parts does the backend consist of? & Name each part of the backend.

The backend consists of the server, the application, and the database.

  • The server is the computer that receives the requests.
  • The application processes the requests - it listens for a request from the client, retrieves the relevant info from the database, and sends back a response to the client.
  • The database contains and organises all the data.

Where does the backend store its data?

In a database, which is an interface that can be accessed by the application to process requests.

What problems do frameworks solve?

Frameworks contain code that is often repeated so that the developer doesn't need to write it all from scratch every time. It also helps structure and create clean, easily readable code.

Name some popular front-end and back-end frameworks.

Django and Flask for Python, Ruby on Rails for Ruby, Express and Meteor for JavaScript.

Describe the process of choosing a framework.

Choosing the correct framework for your application depends on;

  • Ease of use - the language a framework is written in, and your familiarity with it
  • Security
  • Scalability
  • Caching support

What is a "technology stack"?

In order to develop a web application, you need to use a server such as Apache or Nginx, a database such as MongoDB, a programming language (such as Python, Ruby, or PHP), a backend framework (such as Django for Python, or Laravel for PHP), and a front end framework such as React for JavaScript. You also need a host for your server.
E.g. Wordpress is a web application written in PHP.
