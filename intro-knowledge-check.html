Asking for Help

Name at least one thing your question should always include.

Always include your code, and the specific issue you are having e.g. ‘I’m trying to return a string when this condition is reached but I get a syntax error on line 15.” Also include what you have already tried that didn't work.

Describe the “XY Problem”.

The XY problem is when someone needs help with X, and thinks they can figure it out by doing Y - but they don’t know how to do Y either, so they ask for help with Y instead of just asking for help with X.

Describe three attributes of a “help vampire”.

  1. Asking for help when they clearly haven't even tried to solve the issue themselves (ie. being lazy).
  2. Asking for help on something many other people have already asked, instead of just Googling/looking at those questions and answers.
  3. Asking vague or broad questions and providing no code or context (again being lazy).

How Does the Web Work?

What is a network?

A network is a group of computers all connected together via a router.

What is the internet?

The internet is a global structure of networks.

What is an IP address?

An IP or Internet Protocol address is a unique series of numbers that identifies a computer linked to a network. A domain name is a human-readable IP address.

What is a router?

A router is a computer that serves to connect a group of computers together, rather than have each computer individually connect to every other computer.

What is an ISP?

An ISP, or Internet Service Provider, is a company that manages routers that connect to other ISP's routers, enabling customers to access the internet.

What are packets and how are they used to transfer data?

Packets are the smaller chunks of website data that are broken up and sent individually - this allows multiple users to download the same website at once, and it loads much faster.

What is a client?

A client is your computer, phone, smartfridge, or whatever else you use to access the internet.

What is a server?

A server is a computer that stores the files that make up web pages or apps. When a user wants to view a website they download a copy of it from the server to their client.

What is a web page?

A web page is a document that consists of code (such as HTML, CSS, and JavaScript) and assets (images and other media files).

What is a web server?

A web server is a computer that hosts the files that make up a webpage on the internet.

What is a web browser?

A web browsers is software that parses the code that makes up websites and displays them.

What is a search engine?

A search engine is a service that finds other web pages.

What is a DNS request?

A DNS or domain name server request, takes the human readable name of a web address and finds the corresponding numbers of the IP address that point to that page.

Which browser are you currently using?

Google Chrome 100.0 (as of May '22).

In your own words, explain what happens when you run a search on google.com.

When a user enters a search term on google.com, Google’s machines search its index for most relevant results, based on the keywords used, the user’s location, the device they’re using, and hundreds of other factors, in order to provide the user with the best results.

Problem Solving

What are the three stages in the problem solving process?

Understanding the problem, planning how to solve the problem, and breaking the problem down into smaller, subproblems.

Why is it important to clearly understand the problem first?

If you don't understand the problem, you won't know where to start, could waste a lot of time on the wrong things, and confuse yourself even further in the process.

What can you do to help get a clearer understanding of the problem?

Break it down into smaller subproblems, and try to solve those first. "This technique is the cornerstone of problem-solving." - Richard Reis.

What are some of the things you should do in the planning stage of the problem solving process?

Draw a sketch of the interface, draw diagrams, write pseudocode, explain the project to someone else (like 'rubber duck debugging').
Make sure that you fully understand what it is you're trying to achieve, and break it down into steps.

What is an algorithm?

An algorithm is the series of steps taken to accomplish a task.

What is pseudocode?

Pseudocode is the instructions or steps that a program will take to accomplish a task, but it's just written in normal language, for humans to read instead of actual code.

What are the advantages of breaking a problem down and solving the smaller problems?

Looking at one large problem can seem like an insurmountable task. Breaking it down into smaller sub-problems means you can focus on each small part at a time, making things clearer and easier.

Understanding Errors

What are three reasons why you may see a TypeError?

  1. Passing a variable of one type to a function which is expecting a different type.
  2. Trying to modify a value that can't be changed, e.g. reassigning a const
  3. Trying to use a value in an inappropriate way, e.g. using an array method on a string.

What is the key difference between an error and a warning?

A warning will still allow your code to run, but will warn you of a problem. An error will cause your code not to run at all.

What is one method you can use to resolve an error?

Use the debugger in your browser, use console.log, or put a debugger; in your code where the error is occuring. In this way you can follow the steps of your code and see what you actually told it to do, vs what you think you told it to do.

Clean Code

Why is it important to write clean code?

It's important to write clean code so that other developers (and future you) can read it easily and understand what it does.

Name 5 clean code principles previously mentioned

  1. Plan out your code before you write it so it doesn't turn into impossible-to-read spaghetti.
  2. Use indentation, and use it consistently - it will go a long way to making your code easier to understand for the next person (or your future self).
  3. Break up large functions into smaller ones.
  4. Use standard naming conventions - e.g. starting variables with nouns and functions with verbs, using camelCase, and
  5. Don't overuse comments, your code should be reasonably self explanatory.

What is the difference between good comments and bad comments?

Good comments are concise and to the point, allowing the developer reading it to get a quick extra bit of knowledge about that piece of code.
Bad comments are things like leaving a note to self, blaming someone else for writing a piece of code, or being vague.
